Introduction Last updated: 2024-06-23

Lancer’s Hub is a virtual office platform designed to provide flexible, collaborative, and cost-effective virtual office solutions for freelancers, startups, and remote workers in Ethiopia.

Background of Organization

Lancers Hub aims to revolutionize virtual workspaces in Ethiopia by providing a flexible, collaborative, and cost-effective virtual office solution. Drawing inspiration from platforms like Gather.town and Addis Hiwot General Hospital's telemedicine initiatives, Lancers Hub creates a digitally connected environment tailored for freelancers, startups, and remote workers

Statement of the Problem

Freelancers lack dedicated workspaces, which hampers productivity and professionalism. Startups face challenges securing affordable office spaces, and remote workers struggle with isolation and maintaining work-life balance. Traditional office setups limit collaboration and communication, necessitating a virtual solution​.

Objectives of the Project

General Objective:

To develop an immersive virtual office environment tailored to the needs of freelancers, startups, and remote workers in Ethiopia.

Specific Objectives:


Create a customizable virtual workspace with features like video conferencing, screen sharing, and virtual events.

Foster seamless communication and collaboration through proximity-based video streams and shared interactive tools.

Ensure scalability, maintainability, and flexibility through a microservices-based architecture

Methodology and Approach

Data Collection:

Surveys, interviews, and focus groups with target users to understand their needs and preferences.

System Development:

Agile methodology with iterative development cycles to adapt to changing requirements and user feedback.

User-Centered Design (UCD) approach to ensure the platform meets user needs

Tools

Integrated Development Environment (IDE): Visual Studio for coding and debugging.

Game Engine: Unity for building the virtual office environment with C# as the programming language.

Project Management and Collaboration: Git and GitHub for version control, code reviews, and issue tracking.

Scope and Limitation

Scope:

Development of a customizable and interactive 2D virtual office environment.

Integration of spatial audio, video conferencing, and other collaborative tools

Limitations:

Technological constraints due to reliance on evolving technologies like WebRTC and Unity.

Challenges in user adoption and acceptance, especially among those accustomed to traditional work environments.

Infrastructure limitations in Ethiopia affecting performance and accessibility

Significance of the Project

The project offers significant benefits to freelancers, startups, remote workers, and the broader professional community by providing a dynamic, inclusive, and productive virtual workspace. It addresses the challenges of physical office limitations and fosters innovation and growth

Current System

Gather.town is a web-based platform providing virtual spaces for remote teams to collaborate, socialize, and hold meetings. It combines video conferencing with customizable virtual environments, creating an immersive experience akin to a physical office.

Platform Overview

Gather.town offers customizable 2D environments where users navigate with avatars. It uses spatial audio to simulate real-world conversations, allowing users to hear each other based on proximity within the virtual space

User Interface and Experience

The user interface of Gather.town is intuitive, with users navigating using arrow keys or mouse controls. Avatars represent users, making movements and interactions visible to others in real time, fostering a sense of presence and collaboration.

Customization Options

Gather.town provides extensive customization options for designing virtual spaces. Users can create custom maps, upload images for backgrounds, and place objects such as furniture, whiteboards, and interactive elements like games or presentation screens.

Communication and Collaboration Features

Gather.town offers various features including:

  • Spatial audio for realistic proximity-based conversations
  • Video calls for face-to-face interactions
  • Text chat for asynchronous communication
  • Screen sharing for presentations and document collaboration

    • Security and Privacy Measures

    Gather.town encrypts data transmission and offers options for controlling access to virtual spaces. Users can create private spaces for confidential discussions and limit access to authorized participants.

    Integration Capabilities

    Gather.town integrates with other collaboration tools and platforms such as Google Drive, Slack, and Zoom, enabling users to leverage existing workflows and tools within the virtual office environment.

    Scalability and Performance

    Gather.town is designed to scale according to user needs, supporting both small team meetings and large-scale events with hundreds of participants .

    Proposed System

    Lancers Hub aims to provide a virtual office environment leveraging modern technologies for real-time interaction and user engagement.

    The functional requirements include managing user authentication, workspace creation, project management, real-time communication, and integration with external tools.

    1. User Management:
      • Profile management
      • Authentication and authorization
    2. Workspace Management:
      • Creation and organization of virtual workspaces

    3. Project and Task Management:

    4. Communication:
      • Real-time audio and video communication

    5. Integration:
      • Integrating with tools like Google Drive and Slack

    2. Performance:

      Achieve fast response times and efficient processing.

    3. Security:

      Protect user data and system resources from unauthorized access.

    4. Usability:

      Provide an intuitive and easy-to-navigate interface.

    5. Maintainability:

      Ensure the system is easy to maintain and extend over time.

    6. Reliability:

      Ensure the system performs consistently and recovers from failures.

    7. Interoperability:

      Ensure the system integrates with other systems and technologies.

    8. Flexibility:

      Design the system to adapt to changing requirements .

    System Design

    System design details the architectural and technical specifications of the Lancers Hub platform, providing a blueprint for the system's design. It outlines how components interact to meet both functional and non-functional requirements, ensuring the development of a robust, scalable, and maintainable platform.

    Purpose of the System

    The Lancers Hub system aims to create a virtual office environment to facilitate remote work and collaboration among freelancers, startups, and remote workers. It offers a flexible, cost-effective, and immersive workspace.

    EER (Entity-Relationship)

    Defines the entities and attributes in the system:

    • User: UserID, Name, Email, Password, Role, RegistrationDate
    • Workspace: WorkspaceID, Name, Description, CreatedDate, CreatedBy
    • Project: ProjectID, Name, Description, StartDate, EndDate, WorkspaceID
    • Task: TaskID, Title, Description, Status, DueDate, AssignedTo, ProjectID
    • Message: MessageID, Content, SentDate, SenderID, WorkspaceID
    • File: FileID, FileName, FilePath, UploadedDate, UploadedBy, WorkspaceID

    Normalization

    Focuses on organizing the database to reduce redundancy and improve data integrity. It ensures that data is stored efficiently by dividing larger tables into smaller, manageable ones.

    Relational Tables

    Details the relationships between the various tables in the database, ensuring referential integrity and facilitating efficient data retrieval and manipulation.

    OO-Relational Mapping

    Describes how object-oriented models map to relational databases. This includes the transformation of class diagrams into tables, ensuring that the object-oriented design is effectively implemented in the relational database.

    Design Goals

    The goals include:

    1. Scalability: Handle increasing users and data without performance loss.
    2. Performance: Fast response times and efficient processing.
    3. Usability: User-friendly and intuitive interface.
    4. Maintainability: Easy to maintain, update, and extend.
    5. Reliability: Consistent performance and fault tolerance.
    6. Interoperability: Easy integration with other systems.
    7. Flexibility: Adaptable to changing requirements.

    Current Software Architecture

    The current architecture supports a robust virtual office experience:

    1. Frontend: Developed using Unity for 3D rendering and interactivity, with WebRTC for real-time communication and Three.js for 2D graphics.
    2. Backend: Utilizes Firebase for serverless backend services, including a real-time database, authentication, and cloud functions. Node.js and Express.js are used for server-side logic and APIs.
    3. Database: Firebase Realtime Database for data storage and synchronization.
    4. Security: Managed by Firebase Authentication and Security Rules.
    5. Security: Managed by Firebase Authentication and Security Rules.
    6. Communication: WebSockets and WebRTC for real-time data and communication.
